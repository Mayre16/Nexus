✅ FUNCIONALIDADES IMPLEMENTADAS
===========================================================

✅ Búsqueda de Recepciones (2 tipos):
   - Recepción normal (Receptions) → Conduce/Ajuste
   - Compra con Recepción (VendorReceptions) → Proveedor

✅ Campos adicionales en VendorReception:
   - Proveedor (RelationshipName)
   - NCF (Número de Comprobante Fiscal)
   - RNC (FiscalID)
   - No. Factura/Referencia (Reference)
   - ITBIS (TaxAmount)

✅ Productos de VendorReception:
   - Incluye Cost, ExtendedCost (costos)
   - Incluye TaxPercent, TaxAmount (impuestos por línea)

✅ Misma funcionalidad de asignación:
   - Ambos tipos permiten asignar ubicaciones físicas
   - Ambos crean movimientos tipo RECEIPT
   - Ambos incrementan StockUbicacion

📋 ENDPOINTS DISPONIBLES
===========================================================

POST /api/recepciones/buscar
→ Parámetros: {docid, tipo} 
→ tipo: "RECEPTION" o "VEND_REC"
→ Busca en Receptions o VendorReceptions según el tipo

POST /api/recepciones/registrar
→ Registra recepción asignando ubicaciones físicas
→ Funciona con ambos tipos de recepción

GET /recepciones
→ Página HTML del módulo de Recepción
→ Ahora incluye selector de tipo
