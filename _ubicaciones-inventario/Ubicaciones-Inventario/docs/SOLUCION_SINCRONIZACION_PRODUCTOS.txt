   1. Obtiene stock de /api/Stock?LocationID=...&skip=0&take=50
   2. Para cada item con stock > 0:
      ✓ Busca/crea producto en BD (datos básicos)
      ✓ Guarda stock en StockProductoADM
   3. Commit y respuesta rápida al admin ✅
   4. Automáticamente inicia thread en segundo plano ⬇️

ACTUALIZACIÓN EN SEGUNDO PLANO (AUTOMÁTICA):
   1. Obtiene productos de /api/Items?skip=0&take=50 (paginación)
   2. Para cada producto:
      ✓ Si existe → actualiza nombre, SKU, código de barras (solo si cambió)
      ✓ Si no existe → crea producto (sin stock, viene de sync ubicación)
   3. Commit periódico cada 100 productos
   4. NO bloquea sincronización de ubicación
   5. Se ejecuta en background sin timeout ✅
